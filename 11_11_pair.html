pair 可以把两个值"绑"在一起当做一个整体。比如坐标 (x, y)、名字和分数 ("Alice", 85)。使用前需引入 <utility> 头文件（不过 <map>、<vector> 等头文件通常已经间接包含了它，很多时候不需要单独引入）。

pair<string, int> p = {"Alice", 85}；两个不同类型的值打包成一个整体
"Alice"
85
p.first
p.second
两个值的类型可以不同（这里是 string 和 int），但一旦定义好类型就固定了，不能再换。通过 .first 和 .second 分别访问。

11.11.2　常用操作

🔍 访问与比较

C++ · first / second 访问，以及字典序比较
1pair<string, int> p1 = {"Alice", 85};
2pair<int, int> p2 = {3, 4}; // 坐标
3
4// 访问
5cout << p1.first << ": " << p1.second << endl; // Alice: 85
6cout << "x=" << p2.first << ", y=" << p2.second << endl; // x=3, y=4
7
8// 比较：先比 first，first 相等再比 second
9pair<int, int> a = {1, 5}, b = {1, 3};
10cout << (a > b); // 输出：1（true），因为 5 > 3
11
12// C++17 结构化绑定（更直观）
13auto [name, score] = p1; // name = "Alice", score = 85
💡
pair 的默认比较规则：先比 first，first 相等时再比 second。这个特性在竞赛中很常用——比如把 vector<pair<int,int>> 直接 sort()，不用写自定义比较函数，默认就是"先按第一个值排序，相同再按第二个值排序"。
📌
结构化绑定（C++17）：auto [name, score] = p1; 比 p1.first / p1.second 更直观，尤其是配合范围 for 遍历 map 时：for (auto& [key, val] : m) 比 p.first / p.second 可读性好得多。

什么时候用 pair？需要临时打包两个相关的值时：函数要返回两个结果、坐标点、边的两个端点（图论里常见 pair<int,int> 表示一条边）、需要按"主次双关键字"排序的数据等。如果要打包三个以上的值，看下一章的结构体或 C++11 的 tuple。
